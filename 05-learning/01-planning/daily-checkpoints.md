# Daily checkpoints — Reto AI-First Fase 1

## Propósito

Definir el formato único para cerrar una jornada o iteración del reto con información verificable y anexable a `SOUL.md`.

## Formato mínimo

```markdown
1. Avance de hoy: <solo hechos trabajados y verificables>
2. Bloqueos o riesgos: <bloqueos reales o “sin bloqueos identificados”>
3. Siguiente paso: <acción concreta siguiente>
```

## Bloque para anexar a SOUL.md

```markdown
## Checkpoint YYYY-MM-DD

- Avance: <resumen verificable>
- Riesgos/Bloqueos: <riesgos o bloqueos>
- Próximo paso: <siguiente acción>
- Evidencia: <archivos, comandos o resultados revisados>
```

## Reglas

- No inventar avances.
- Si falta información, indicarlo explícitamente.
- No registrar decisiones de stack aquí; esas pertenecen a `05-learning/01-planning/ai-first-challenge-tech-stack.md` y `05-learning/01-planning/governance.md`.
- No registrar procedimientos reutilizables aquí; esos pertenecen a skills o documentación específica.
- Mantener cada checkpoint breve y trazable.
