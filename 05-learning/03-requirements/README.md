# Requirements

> Carpeta para convertir el entendimiento del proyecto en requerimientos accionables antes de escribir código.

## Purpose

Esta carpeta contiene el entendimiento funcional del Portal de Convocatorias Públicas y las Historias de Usuario (HU) derivadas.

## Files

- `05-learning/03-requirements/project-understanding.md` — entendimiento del problema, usuarios, alcance y supuestos.
- `05-learning/03-requirements/user-stories.md` — Historias de Usuario priorizadas.
- `05-learning/03-requirements/acceptance-criteria.md` — criterios de aceptación transversales y por HU.
- `05-learning/03-requirements/scope-coverage.md` — matriz end-to-end de alcance del reto contra HU.

## Rule

Ninguna HU debe pasar a implementación si no tiene:

- objetivo claro,
- usuario/actor,
- criterio de aceptación,
- capa o capas afectadas,
- impacto revisado con `05-learning/00-traceability/change-guardian.md`.
